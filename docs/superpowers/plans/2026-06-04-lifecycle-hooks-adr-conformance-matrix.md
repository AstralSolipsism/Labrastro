# Lifecycle Hooks ADR Runtime Conformance Matrix

> **For agentic workers:** This matrix is the acceptance gate for `Lifecycle Hooks ADR Runtime Conformance Closure`. Do not treat a bug fix, focused green test, or local handler path as completion unless the relevant rows are proven green by current-state evidence.

**Goal:** Prove ADR 0006 lifecycle hook contracts are implemented in real runtime paths, not only schema, dashboard, metadata, or local tests.

**Architecture:** Validate from the ADR outward. Each row must name the ADR requirement, implementation entry points, test evidence, fail-closed or fail-open behavior, and user/audit visibility. Rows marked `GAP` or `EVIDENCE MISSING` are not complete.

**Tech Stack:** Python lifecycle domain, Agent/ToolExecutor, PermissionGateway, AgentRun control plane, SessionRun projection, admin/dashboard APIs, pytest and frontend Vitest suites.

---

## Status Legend

- `GREEN`: Current code and tests prove the ADR requirement for the full stated scope.
- `GAP`: Current evidence contradicts the ADR or proves only a narrower scope.
- `EVIDENCE MISSING`: Implementation may exist, but current evidence is too weak or indirect.
- `NOT WIRED`: ADR event or surface is cataloged but not connected to real runtime dispatch.

No row may be marked `GREEN` from intent, naming, dashboard visibility, or metadata storage alone.
